# 京东

## 支持场景

- `search`
- `product_detail`
- `shop_hot_products`

## 当前状态

- 已完成平台拆分和维护文档补齐。
- 2026-04-04 实测：已补“先走 `hk.jd.com` 首页，再输入关键词”的交互搜索流程。
- 当前首页交互后仍会跳到 `passport.jd.com` 登录页，`search` 场景暂时稳定识别为 `login_required`。

## 目标页面

- 搜索页：优先 `https://hk.jd.com/` 首页交互，翻页兜底仍保留 `https://search.jd.com/Search?keyword={keyword}&page={page}`
- 详情页：商品详情页
- 热门商品页：搜索列表、店铺商品列表

## 选择器策略

- 搜索页优先依赖 `.gl-item[data-sku]`。
- 标题、价格、店铺等沿用京东传统 PC 结构。
- `data-sku` 是目前最值得优先保留的稳定字段。
- 搜索动作已拆到场景 `preparePage`，后续换入口时只需要改平台目录。

## 风险与验证码

- 可能出现登录、验证、访问频率限制。
- 命中风险后当前不会持续重试，会直接结束本次任务。

## 维护方式

1. 先确认 `.gl-item[data-sku]` 是否还存在。
2. 如果首页交互后还是进入登录页，优先判断环境、Cookie 或 IP 问题，而不是马上改 DOM 选择器。
3. 如果页面切到新版卡片结构，先保住 `data-sku` 和商品链接，再补标题价格。
4. 如果列表懒加载明显，可考虑平台层补滚动逻辑。
