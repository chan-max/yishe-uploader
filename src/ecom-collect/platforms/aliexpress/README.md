# AliExpress

## 支持场景

- `search`
- `product_detail`
- `shop_hot_products`

## 当前状态

- 已完成平台目录拆分和选择器整理。
- `search` 已做真实浏览器回归并可返回有效结果。
- `product_detail / shop_hot_products` 目前仍属于可运行脚手架，后续需要继续验证。

## 目标页面

- 搜索页：`https://www.aliexpress.com/wholesale?SearchText={keyword}&page={page}`
- 详情页：商品详情页 `/item/`
- 热门商品页：店铺首页或活动商品列表

## 选择器策略

- 优先依赖 `/item/` 链接特征确定商品节点。
- 搜索页标题优先抓 `[role="heading"] / h3.lw_k4`。
- 搜索页价格优先抓 `div.lw_el / div.lw_kt`，避免把折扣、销量混进标题。
- 其他字段仍保留包含式 class 兜底，适应多语言和动态样式。
- 平台目录里额外使用 `normalizeRecord` hook，把搜索卡片中的“现价 / 原价 / 折扣”做一次轻量拆分，当前保留首个现价。

## 风险与验证码

- 海外站常见弹窗、地区确认、货币切换。
- 如果页面跳登录、访问限制或安全验证，通用风控识别会直接返回失败。

## 维护方式

1. 先确认结果页里商品链接是否仍然保留 `/item/` 特征。
2. 如果平台切换到更重的前端渲染，优先观察卡片根节点是否需要滚动后才出现。
3. 如果价格拆成多个片段，优先补充价格候选选择器，不要过早改根节点规则。
