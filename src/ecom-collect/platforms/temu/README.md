# Temu

## 支持场景

- `search`
- `product_detail`
- `shop_hot_products`

## 当前状态

- 已完成模块化和初版选择器整理。
- 2026-04-04 实测：搜索结果页会直接跳到 `login.html`。
- 通用层已补上“无结果后二次风险识别”，当前会稳定返回 `login_required`，不再误报成“未采集到有效结果”。

## 目标页面

- 搜索页：`https://www.temu.com/search_result.html?search_key={keyword}&page={page}`
- 详情页：商品详情页
- 热门商品页：店铺商品列表或专题商品流

## 选择器策略

- 以 `a[href*="-g-"]` 作为商品链接主特征。
- `data-testid` 作为补充兜底。
- 标题、价格、店铺名均优先使用包含式选择器，避免绑定动态 hash class。

## 风险与验证码

- Temu 经常弹注册/登录层。
- 命中登录或风控时，当前策略是快速退出并返回风险类型，不在平台内部死循环处理。
- 当前真实阻塞点是搜索入口要求登录，不是选择器失效。

## 维护方式

1. 优先确认商品链接仍然带 `-g-`。
2. 如果页面先显示弹窗，后续可在平台 hook 里加“关闭弹窗”逻辑，不要污染公共流程。
3. 如果结果页必须滚动加载，可在平台层增加场景预处理。
