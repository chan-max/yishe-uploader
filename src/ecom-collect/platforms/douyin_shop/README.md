# 抖音店铺

## 支持场景

- `search`
- `product_detail`
- `shop_hot_products`

## 当前状态

- 已完成目录拆分和选择器初版整理。
- 2026-04-04 实测：`jingxuan` 首页存在登录覆盖层，会拦截搜索框点击。
- 已在平台场景里补了覆盖层移除和首页搜索交互，但当前仍会被登录流拦住，`search` 场景暂时稳定识别为 `login_required`。

## 目标页面

- 搜索页：当前从 `https://www.douyin.com/jingxuan` 首页交互进入
- 详情页：商品详情页
- 热门商品页：店铺商品流

## 选择器策略

- 优先尝试 `/product/` 链接和 `data-e2e` 属性。
- `data-e2e` 通常比动态 class 更稳定，适合作为优先候选。
- 搜索交互当前优先依赖 `data-e2e="searchbar-input"` 和 `data-e2e="searchbar-button"`。

## 风险与验证码

- 常见问题：登录页、扫码登录、滑块、访问限制。
- 当前命中风险后会直接退出，避免任务长期卡住。
- 这类页面很容易同时出现“登录”和“验证码登录”字样，通用风险识别已调整为优先判定 `login_required`。

## 维护方式

1. 先判断 `jingxuan` 首页是否还能正常进入。
2. 如果搜索框被覆盖层拦截，优先修覆盖层选择器，再看搜索结果页。
3. 如果进入登录流，继续在平台目录里处理，不要把抖音特例写进公共流程。
4. DOM 变动时优先重新找 `data-e2e` 和稳定链接模式。
